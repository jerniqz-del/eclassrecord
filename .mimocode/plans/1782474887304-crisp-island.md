# Class Analysis Feature

## Goal
Add a "Reports" button to each class card on the Dashboard that opens a comprehensive class analysis view. The analysis shows per-assessment statistics (mean, median, mode, performance level distribution) and per-learner performance details (term grades, component breakdown, badges, rank). Uses a term selector to switch between Term 1/2/3/Summary.

---

## Architecture

### New File
- **`src/renderer/js/class-analysis.js`** вҖ” All computation + rendering logic + CSS chart builders for the analysis modal (~500-600 lines)

### Modified Files
- **`src/renderer/index.html`** вҖ” Add analysis modal HTML + "Reports" button in dashboard class cards
- **`src/renderer/js/dashboard.js`** вҖ” Wire up the "Reports" button click handler
- **`src/renderer/js/app.js`** вҖ” Include the new JS file in script loading

---

## Data Flow

```
User clicks "Reports" on class card
  вҶ’ showClassAnalysisModal(assignmentId)
    вҶ’ computeClassAnalysis(a, term)
      вҶ’ Per-assessment: mean, median, mode, pass rate, performance distribution
      вҶ’ Per-learner: term grades, component scores, rank, badge
    вҶ’ renderClassAnalysis(a, term, analysisData)
      вҶ’ Returns HTML string for the modal body
```

## Implementation Details

### 1. `src/renderer/js/class-analysis.js` вҖ” New file

#### `computeClassAnalysis(a, term, mapePart)`
Main computation function. Returns:
```js
{
  assessments: [{
    assessment: AssessmentObject,
    scores: number[],           // all learner scores for this assessment
    mean: number,               // average score
    median: number,             // middle value
    mode: number|null,          // most frequent score (null if no mode)
    min: number,
    max: number,
    passRate: number,           // % scoring >= 75% of HPS
    performanceLevel: {         // count per level
      advanced: number,         // >=90% of HPS
      proficient: number,       // >=75% but <90%
      developing: number,       // >=50% but <75%
      beginning: number         // <50%
    },
    distribution: [number, number, number, number, number]  // count per 20% range: [0-20%, 20-40%, 40-60%, 60-80%, 80-100%]
  }],
  learners: [{
    learner: LearnerObject,
    termGrade: number|string|null,
    initialGrade: number|null,
    ww: { raw, max, ps },
    pt: { raw, max, ps },
    exam: { raw, max, ps },
    remarks: string,            // 'Passed' | 'Failed' | 'For Intervention'
    rank: number,               // 1-based rank among classmates
    percentile: number          // percentile rank
  }],
  classStats: {
    mean: number,               // class mean of term grades
    median: number,             // median of term grades
    mode: number|null,          // mode of term grades
    passRate: number,           // % passing
    distribution: {             // performance level counts
      advanced: number,
      proficient: number,
      developing: number,
      beginning: number
    },
    gradeDistribution: [number, number, number, number, number]  // [<75, 75-79, 80-84, 85-89, 90-100]
  }
}
```

**Statistical helper functions:**
- `calcMean(arr)` вҖ” sum / count
- `calcMedian(arr)` вҖ” sorted middle value (or average of two middle values)
- `calcMode(arr)` вҖ” most frequent value, null if all unique
- `calcPercentile(arr, value)` вҖ” percentile rank of a value in the array

#### `renderClassAnalysisModal(a, term)`
Returns the full modal HTML string with:
- **Header**: Class name (Grade X - Section), Subject, School Year
- **Term selector**: 4 buttons (Term 1, Term 2, Term 3, Summary)
- **Class Summary card**: Mean, Median, Mode, Pass Rate, Performance Distribution (4 bars or badges)
- **Assessment Analysis table**: One row per assessment with mean, median, mode, pass rate, performance level breakdown
- **Learner Performance table**: Ranked list with name, sex, term grade, component breakdown (WW/PT/Exam PS), remarks badge, rank

#### `renderClassSummary(stats)`
Renders the class-level statistics card with:
- 4 stat boxes: Mean, Median, Mode, Pass Rate
- **Performance Distribution Chart** вҖ” CSS horizontal stacked bar with colored segments (advanced=green, proficient=teal, developing=amber, beginning=red) and labels/counts

#### `renderAssessmentAnalysisChart(assessments)`
**CSS bar chart** showing mean/median/mode per assessment as grouped vertical bars:
- Each assessment gets a cluster of 3 bars (mean=teal, median=blue, mode=gray)
- Y-axis: score scale (0 to max HPS)
- X-axis: assessment labels (WW 1, PT 1, etc.)
- Built with pure CSS `div` elements with `height` percentages вҖ” no charting library

#### `renderScoreDistributionChart(assessments)`
**CSS histogram** for each assessment showing score distribution:
- X-axis: score ranges (0-20%, 20-40%, 40-60%, 60-80%, 80-100% of HPS)
- Y-axis: number of learners in each range
- Built with CSS flex columns with dynamic heights

#### `renderLearnerGradeDistribution(learners)`
**CSS bar chart** showing grade distribution across the class:
- X-axis: grade ranges (Below 75, 75-79, 80-84, 85-89, 90-100)
- Y-axis: number of learners
- Color-coded: red (below 75), amber (75-79), teal (80-84), green (85-89), dark green (90-100)

#### `renderAssessmentAnalysisTable(assessments)`
Table with columns:
| Assessment | HPS | Mean | Median | Mode | Pass Rate | Advanced | Proficient | Developing | Beginning |

#### `renderLearnerPerformanceTable(learners)`
Ranked table with columns:
| Rank | Learner | Sex | WW PS | PT PS | Exam PS | Initial Grade | Term Grade | Remarks |

- Rows sorted by term grade descending
- Pass/Fail badges in remarks column
- Highlight top 3 rows (gold/silver/bronze styling)

### 2. `src/renderer/index.html` вҖ” Modal + button

#### Add "Reports" button in dashboard class cards
In `renderDashboardOverview()` output (generated by dashboard.js), add a "Reports" button next to existing card actions. The button HTML will be injected by dashboard.js.

#### Add analysis modal HTML
Add a new modal overlay at the bottom of `<body>`:
```html
<div id="classAnalysisModal" class="modal-overlay" style="display:none; z-index: 10002;">
  <div class="modal modal--wide" style="max-width: 900px; max-height: 90vh; display: flex; flex-direction: column;">
    <div class="modal__title">Class Analysis</div>
    <div class="modal__body" style="flex: 1; overflow-y: auto; padding: var(--space-4);" id="classAnalysisBody">
      <!-- Populated by renderClassAnalysisModal() -->
    </div>
    <div class="modal__actions">
      <button class="btn btn-ghost btn-sm" onclick="closeClassAnalysisModal()">Close</button>
    </div>
  </div>
</div>
```

### 3. `src/renderer/js/dashboard.js` вҖ” Wire up button

In `renderDashboardOverview()` or the per-card rendering section, add a "Reports" button:
```html
<button class="btn btn-olive btn-sm" onclick="showClassAnalysisModal('${esc(a.id)}')">
  <svg class="btn-icon" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">
    <path d="M18 20V10"></path><path d="M12 20V4"></path><path d="M6 20v-6"></path>
  </svg>
  Reports
</button>
```

### 4. `src/renderer/js/app.js` вҖ” Script include

Add `<script>` tag for `class-analysis.js` in `index.html` (after existing JS includes), or add the file to the HTML's script list.
